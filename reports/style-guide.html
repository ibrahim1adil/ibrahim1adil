Overall Formatting

The site colors are mostly blue and teal gold accent line. background color is a light blue-gray (#f4f6f9) and most of the text is dark gray (#222). White for the project cards.

font for the whole site is Segoe UI with backups like Tahoma, Geneva, and Verdana. The text size is the browser default and headings are bigger.

The main colors I used:

  • #f4f6f9 - page background
  • #1a4b8c - main blue (used in the header/nav buttons)
  • #0d6e6e - teal (used for the h2 headings/the active nav link)
  • #f0a500 - gold accent line on top of the header
  • #222222 - body text
  • #888888 - footer text

Site Header

The header is an h1 with class .site-header. uses the main blue (#1a4b8c) at 2.2rem with a 3px blue line under it and a 5px gold line on top. On phones it goes down to 1.5rem so it can fit better.

Navigation Links

The nav links are in the nav element. They're blue rounded buttons with white text. When you hover over a link it turns a lighter blue, and the link for the page you are on's teal. The nav uses Flexbox so the buttons line up in a row, and on phones so they stack on top of each other instead.

Content Section

The content goes inside a div called .page-wrapper that is centered on the page with a max-width of 1000px so it doesn't get too long on bigger screens. There is 20px of padding around the content (15px on tablets).

Section headings (h2) are teal with a teal line on the left. Project cards are white with a small shadow and rounded corners. I put them in a div called .project-grid that uses Flexbox so the cards sit next to each other on big screens and stack on phones.

Site Footer

The footer has small gray text (0.8rem, color #888) with a thin line on top. It is centered. Because the body uses display: flex with flex-direction: column and the page-wrapper has flex: 1, the footer stays at the bottom of the page even when there isn't much content.

Layout and Responsive Design

I made a new file called layout.css in the _css folder that has the layout rules for the site. It is linked on every page after format.css. The layout.css file has more than 10 rules for the header, nav, main, project grid, skills list, footer, and images, plus two media queries.

I used three responsive web design ideas:

  1. Media queries at two breakpoints. At 800px the padding gets smaller. At 500px the nav links stack into a column and the project cards go to one column.
  2. Flexbox with flex-wrap on the nav and the project grid so the items move to a new row on their own when the screen gets smaller.
  3. Flexible images with img { max-width: 100%; height: auto; } so any image will shrink to fit and not overflow the page.
